# Structurizr Workspace

This folder contains the baseline Structurizr DSL workspace for the repository architecture.

## Entry Point

- `workspace.dsl`

## Structure

- `model/`
  - people, software systems, containers, components
- `views/`
  - system context, container, component, dynamic, and deployment views
- `styles/`
  - shared view styling

## Working Rule

Use this workspace as the structured architecture source when a feature materially changes:

- system boundaries
- containers
- major components
- important workflows
- deployment topology

Keep markdown architecture documents in `specs/architecture/` as short explanatory companions rather than duplicating every DSL detail here.
