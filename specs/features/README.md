# Feature Specs

Store feature packages here using the canonical templates in [templates](../../templates/README.md).

Each feature package should include:

- source request artifacts when relevant
- feature specification
- supporting design artifacts when relevant
- test and validation artifacts
- traceability links to tests and implementation

Active governing packages should follow the capability-based naming convention `FEAT-<AREA>-<ID>-<short-name>`.

Narrow enhancement-only packages should be absorbed into the appropriate governing package so the active feature inventory remains compact and capability-oriented.
