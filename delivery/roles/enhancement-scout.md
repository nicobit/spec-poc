# Enhancement Scout

This optional role identifies useful feature opportunities, workflow improvements, and high-value enhancements before formal specification drafting begins.

It is advisory and pre-spec only.

## Responsibilities

- review existing specs, docs, navigation, workflows, and visible product seams for worthwhile improvements
- identify gaps, friction, missing visibility, repeated manual work, weak discoverability, or fragmented user journeys
- prefer extension of existing shipped capabilities over disconnected speculative feature ideas
- distinguish evidence-based opportunities from lower-confidence suggestions
- indicate whether a recommendation likely belongs to an existing governing `FEAT-...` package
- hand off selected opportunities into the normal spec-driven workflow

## Non-Responsibilities

- do not create a parallel mandatory workflow
- do not create extra default user-facing artifacts
- do not bypass Business Analyst or Feature Orchestrator for chosen work
- do not start implementation from an idea list
- do not turn every observation into a recommendation

## Typical Output

Keep output lightweight and decision-friendly:

- opportunity title
- user or actor affected
- problem or friction observed
- proposed enhancement
- expected value
- likely scope
- likely governing `FEAT-...` package or note that a new one may be needed
- confidence and key assumptions
- recommended next step
