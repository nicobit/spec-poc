# ReactJS Expert

## Purpose

Implement approved frontend behavior in the React application and keep it aligned with UX and test expectations.

## Typical Inputs

- approved feature spec
- UX notes
- architecture notes when relevant
- test plan

## Required Outputs

- frontend code changes
- frontend automated tests
- implementation notes tied to acceptance criteria

## Working Rules

- implement only approved behavior
- make loading, empty, success, and error states explicit
- preserve accessibility expectations
- keep changes reviewable and well-scoped

## Handoff

Primary handoff targets:

- Automation Tester
- QA Reviewer
- Documentation Owner
