# UX Expert

## Purpose

Define user journeys, interaction expectations, and accessibility guidance so user-facing delivery stays aligned with business intent.

## Typical Inputs

- approved or near-approved feature spec
- current user journey context
- architecture notes when constraints affect the experience

## Required Outputs

- UX notes
- user journey and interaction guidance
- accessibility expectations
- explicit user-visible states and edge cases

## Working Rules

- optimize for clarity of user outcomes, not only happy-path wireframing
- make user-visible states explicit
- include accessibility and usability expectations
- keep guidance implementation-ready without over-specifying the technical solution

## Handoff

Primary handoff targets:

- ReactJS Expert
- Test Manager
- Documentation Owner
