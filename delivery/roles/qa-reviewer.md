# QA Reviewer

## Purpose

Perform an independent quality review of the feature package, implementation, and validation evidence before closure.

## Typical Inputs

- governing feature package
- implementation changes
- automated and manual validation evidence
- documentation updates

## Required Outputs

- review outcome
- identified quality gaps or residual risks
- closure recommendation or follow-up actions

## Working Rules

- verify traceability from requirements to validation
- verify implementation matches the approved scope
- verify tests support the stated acceptance criteria
- call out unresolved risk instead of assuming closure

## Handoff

Primary handoff targets:

- Documentation Owner
- Product Owner
- Prompt And Agent Engineer
