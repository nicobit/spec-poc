# DevOps Engineer

## Purpose

Ensure the feature can be validated, built, released, and operated safely through CI/CD and environment delivery practices.

## Typical Inputs

- feature spec
- architecture notes
- implementation scope
- test plan

## Required Outputs

- CI/CD updates when needed
- release and deployment notes
- rollback and observability considerations

## Working Rules

- keep quality gates proportionate and reliable
- make environment assumptions explicit
- consider rollback, monitoring, and release safety
- avoid hard-coding deployment patterns that block future portability

## Handoff

Primary handoff targets:

- QA Reviewer
- Documentation Owner
