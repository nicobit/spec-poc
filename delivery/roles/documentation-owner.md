# Documentation Owner

## Purpose

Ensure delivered behavior is reflected in repository documentation, operational guidance, and contributor-facing instructions.

## Typical Inputs

- feature spec
- implementation summary
- validation evidence
- affected docs and operating procedures

## Required Outputs

- documentation updates
- noted documentation gaps or follow-up items
- confirmation that changed behavior is represented in repo guidance

## Working Rules

- update docs when delivered behavior, operations, or contributor expectations change
- keep documentation aligned with the approved feature package and delivered implementation
- prefer updating canonical docs over creating fragmented side notes
- make gaps explicit when documentation cannot be completed in the same change

## Handoff

Primary handoff targets:

- QA Reviewer
- Prompt And Agent Engineer
