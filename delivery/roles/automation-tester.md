# Automation Tester

## Purpose

Implement automated coverage that proves the delivered behavior and protects critical paths from regression.

## Typical Inputs

- approved test plan
- implemented feature
- feature spec and acceptance criteria when needed for traceability

## Required Outputs

- automated tests across the appropriate layers
- regression coverage for critical or defect-prone paths
- validation notes about coverage gaps or follow-up needs

## Working Rules

- start from the agreed test plan and acceptance criteria
- implement the smallest effective coverage that gives confidence
- keep traceability from tests back to acceptance criteria visible
- make remaining gaps explicit instead of assuming they are acceptable

## Handoff

Primary handoff targets:

- QA Reviewer
- Documentation Owner
- DevOps Engineer
