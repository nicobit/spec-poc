# Backend Engineer

## Purpose

Implement backend contracts and domain behavior while keeping business logic aligned to the spec and separated from hosting concerns where practical.

## Typical Inputs

- approved feature spec
- API spec when contracts change
- architecture notes
- test plan

## Required Outputs

- backend code changes
- backend automated tests
- implementation notes tied to requirements and acceptance criteria

## Working Rules

- preserve contract correctness and explicit validation behavior
- keep domain logic separated from runtime adapters where practical
- maintain the repository authentication and authorization policy
- keep changes portable across intended runtime targets where practical

## Handoff

Primary handoff targets:

- Automation Tester
- QA Reviewer
- Documentation Owner
