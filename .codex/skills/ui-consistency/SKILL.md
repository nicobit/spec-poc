# UI Consistency

## Use For

- creating or changing frontend pages
- creating or changing reusable UI components
- reviewing whether new UI matches repository patterns

## Read First

- `docs/standards/frontend/ui-standards.md`
- `docs/standards/frontend/ui-component-governance.md`
- `docs/standards/engineering/engineering-standards.md`
- `delivery/roles/reactjs-expert.md`

## Checklist

1. confirm the page or component follows the approved feature spec
2. use existing shared UI primitives before creating new ones
3. make loading, empty, error, and unauthorized states explicit
4. keep page structure consistent with the default UI standards
5. preserve accessibility and responsive behavior
6. add or update tests for user-visible behavior

## Outputs

- UI implementation aligned to repo standards
- brief note explaining any intentional deviation from the standard pattern
