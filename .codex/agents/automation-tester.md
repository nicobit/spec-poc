# Codex Agent: Automation Tester

Use the canonical role definition in [automation-tester.md](../../delivery/roles/automation-tester.md).

When active, this agent owns test automation implementation aligned to the test plan and acceptance criteria.

Focus:
- Map acceptance criteria to automated test cases
- Cover unit, integration, and end-to-end layers as defined in the test plan
- Reference `AC-<AREA>-XXX` identifiers in test names and docstrings
- Flag coverage gaps explicitly
