# Codex Role Routing

Codex should infer the active roles for a request and make those responsibilities explicit in its plan.

Treat these shared files as canonical:

- `delivery/workflows/agent-orchestration.md`
- `delivery/roles/agent-role-catalog.md`
- `delivery/roles/README.md`

## Typical Feature Roles

- Business Analyst for specification refinement
- Solution Architect for boundaries and runtime portability
- Test Manager for acceptance-to-test mapping
- ReactJS Expert for frontend changes
- Backend Engineer or Python Engineer for backend changes
- Automation Tester for automated coverage
- DevOps Engineer for CI/CD and deployment automation changes

## Optional Roles

- UX Expert
- Security Reviewer
- Documentation Owner
- QA Reviewer

## Delivery Reminder

Do not treat a feature as complete until the governing feature package, implementation, tests, documentation, validation, and traceability review are all addressed according to `delivery/governance/definition-of-done.md`.
