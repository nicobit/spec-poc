# UI Consistency Review

Use this command when creating or changing pages or reusable UI components.

Read first:

- `docs/standards/frontend/ui-standards.md`
- `docs/standards/frontend/ui-component-governance.md`
- `docs/standards/engineering/engineering-standards.md`

Then:

1. identify the approved feature spec and affected page or component
2. check whether shared UI or shared widgets already cover the need
3. verify loading, empty, error, and unauthorized states are explicit
4. verify accessibility and responsive behavior
5. call out any intentional deviation from the default UI standards
